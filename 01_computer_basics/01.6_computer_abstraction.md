# 📘 הפשטת מחשב ומעברי שפה – זיכרון, Bytecode ופירוש פייתון

## ✨ הסבר
כדי להבין איך הקוד שאנחנו כותבים בפייתון באמת פועל, צריך להכיר את עקרון ההפשטה בעולם המחשבים. 

בין מה שאנחנו רואים – לקוד שרץ בפועל – יש הרבה שכבות: תרגום משפה עילית לשפה מכונה, שימוש בזיכרון, ועבודה של רכיבי החומרה. 

נלמד איך זה עובד – ומה באמת קורה כשמריצים `print("Hello")`.

## 🧠 מה נלמד בפרק הזה?
- מהי הפשטה (Abstraction) ואיך מחשב בנוי בשכבות
- שלבי הריצה של תוכנית פייתון – מהקוד ועד לשפה מכונה
- מהו Bytecode ולמה הוא חשוב
- תפקידי ה־RAM, Registers וה־CPU
- איך Garbage Collector של פייתון מנהל זיכרון

---

## 🔍 הסברים ודוגמאות

### 🧱 מהי הפשטה במחשב?

> מחשב בנוי בשכבות – כל שכבה "מסתירה" את המורכבות שמתחתיה

| שכבה           | דוגמה                         |
|----------------|-------------------------------|
| שפת משתמש      | Python, Java                  |
| Bytecode        | קוד ביניים (`.pyc`)          |
| Interpreter     | Python VM (CPython)          |
| שפת מכונה       | קוד בינארי (010101)           |
| חומרה (CPU)     | מעבד, זיכרון, דיסק            |

---

### 📄 איך פייתון רץ בפועל?

1. אתה כותב קוד:  
```python
print("Hi Dvir")
````

2. המפרש (Interpreter) מתרגם אותו ל־**Bytecode**
3. ה־Python Virtual Machine (PVM) מפעילה את ה־Bytecode
4. מתבצעת קריאה לפונקציות בלולאת פירוש
5. ה־CPU מפעיל פקודות → המידע נכנס לזיכרון → התוצאה מופיעה במסך

---

### 💽 מהו Bytecode?

* קוד ביניים בין פייתון לשפת מכונה
* נשמר בקבצים `__pycache__/*.pyc`
* רץ מהר יותר מהקוד הרגיל, כי הוא כבר תורגם

---

### 🧠 תפקיד ה־RAM, Registers וה־CPU

| רכיב      | תיאור                                       |
| --------- | ------------------------------------------- |
| Registers | אחסון מיידי בתוך ה־CPU (מהיר מאוד)          |
| RAM       | זיכרון העבודה הפעיל של התוכנית              |
| CPU       | מבצע את הפקודות – כולל fetch-decode-execute |

🔁 פקודה → מובאת מהזיכרון → מפורשת → מתבצעת

---

### ♻️ Garbage Collector

* פייתון מנהלת זיכרון באופן אוטומטי
* כשאין שימוש במשתנה – הוא נמחק מהזיכרון
* זה נעשה בעזרת **Reference Counting**
* אם אין הפניות → נמחק

```python
a = [1, 2, 3]
b = a
del a  # עדיין קיים בזכות b
```

---

## 💡 טיפים מתקדמים

* Bytecode לא זהה בין גרסאות פייתון – ⚠️ אל תנסה להריץ `.pyc` מגרסה אחרת
* אם אתה בונה מערכת קריטית – דע איפה הדברים נשמרים בזיכרון
* `gc` הוא מודול בפייתון שמאפשר לך לבדוק ולנהל את ניהול הזיכרון בעצמך
